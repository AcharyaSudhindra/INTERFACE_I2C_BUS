Abstract: This report presents the complete design and verification of an I²C bus master controller and companion behavioral slave models, implemented in Verilog HDL and simulated on Xilinx Vivado xSim. The design demonstrates standard-mode (100 kHz) I²C communication between an FSM-based master and two parameterised peripherals — a simulated OLED display (0x3C) and an EEPROM (0x50). 1 I²C Protocol Overview

I²C (Inter-Integrated Circuit) is a synchronous, multi-master, multi-slave serial bus requiring only two bidirectional open-drain signals. Devices share SDA and SCL; the wired-AND topology holds both lines HIGH (via pull-up resistors) when no device drives them LOW.

START Condition
SDA falls LOW while SCL is HIGH — marks the beginning of a transaction.
STOP Condition
SDA rises HIGH while SCL is HIGH — releases the bus after a transaction.

2 i2c_master.v — Low-Level Protocol Engine

The master encapsulates the complete I²C frame generator. Its clock divider converts the 50 MHz system clock into 100 kHz I²C clock pulses, then resolves each cycle into 4 internal phases to guarantee setup/hold timing.

Phase 0
SCL = 0
Master sets SDA
Phase 1
SCL ↑ (rises)
Clock goes HIGH
Phase 2
SCL = 1
Data sampled here
Phase 3 — SCL ↓: Clock falls; master/slave prepares next bit. SDA only changes during SCL-LOW, guaranteeing stability during SCL-HIGH per the I²C specification. 
// Phase clock divider: 50 MHz system clock → 100 kHz I²C → 4 phases
localparam DIVIDER = (INPUT_CLK_FREQ / I2C_FREQ) / 4;  // = 125 cycles/phase
// SDA tri-state: master drives only when sda_dir == 1, else releases to Hi-Z
assign sda = (sda_dir == 1'b1) ? sda_out : 1'bz;

3 top_controller.v — Application Layer FSM

Orchestrates two complete I²C transactions via a 10-state FSM, driving the i2c_master through enable/rw/addr/data_in signals, then waiting on busy and ack_error.

Tx 1 — OLED Write (0x3C)
Sets rw=0, loads oled_wdata=0xA5, pulses enable. Waits for busy to deassert with no ACK error.
Tx 2 — EEPROM Read (0x50)
Writes pointer byte eeprom_ptr=0x00, then re-issues with rw=1; captures returned byte into eeprom_data.
A 16-bit INTER_TX_DELAY counter inserts quiet time between transactions. In simulation: 200 cycles (4 µs). In hardware: 65 535 cycles (~1.3 ms at 50 MHz).

4 Behavioral Slave Models

Two simulation-only behavioral slaves are provided, both parameterised by SLAVE_ADDR (7-bit) and READ_DATA (8-bit). The critical design separates sampling and driving into distinct clock-edge domains to eliminate race conditions:

posedge SCL → Sample only
shift_reg ← {shift_reg[6:0], sda}
SDA is guaranteed stable during SCL-HIGH per the I²C spec.
negedge SCL → Drive / Advance state
After the data window closes, the slave pulls SDA for ACK or drives READ data — no false START/STOP edges.

Testbench — tb_i2c_system.v

Instantiates the full virtual system: pull-up resistors model the open-drain bus; two slave instances share sda/scl; a 2 ms watchdog prevents hangs; self-checking assertion verifies eeprom_data === 8'h42.

5 Engineering Challenges & Bug Resolutions

BUG #1 Floating Bus → Immediate NACK on First Transaction RESOLVED

● Symptom: Master addressed 0x3C but received NACK instantly — bus never responded.

● Root Cause: No slave modules were instantiated. SDA stayed HIGH via the pull-up (HIGH in the ACK slot = NACK). The master was communicating with an empty bus.

● Fix: Added oled_slave and eeprom_slave instantiations onto the shared sda/scl wires. ACK pulses appeared immediately after.

BUG #2 Bit-Shift Race Condition → Wrong Address Decoded (0x78 instead of 0x3C) RESOLVED

● Symptom: Slave reported address 0x78 instead of 0x3C — exactly one left-shift away.

● Root Cause: The original slave sampled sda on negedge scl. The master also changes SDA shortly after the falling edge, causing a Verilog delta-cycle race: the slave captured the wrong (next) bit, shifting every address by one.

● Fix: Moved data sampling strictly to posedge scl — the only point where I²C guarantees SDA is stable. State transitions kept on negedge scl to avoid generating false START/STOP conditions.

BUG #3 Multi-Driver Error — Vivado xelab Fatal: VRFC 10-529 RESOLVED

● Symptom: xelab refused to elaborate with "concurrent assignment to a non-net is not permitted".

● Root Cause: is_addressed and bit_count were assigned inside both always @(negedge sda) (START detector) and always @(negedge scl) (main FSM). Verilog forbids one reg being driven from two independent always blocks.

● Fix: Introduced single-bit start_seen / stop_seen flags, each written by exactly one always block. The main FSM reads the flags without driving registers from multiple domains — clean elaboration.
